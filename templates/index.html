About the data
Every signal shown is grounded in real, verifiable FDA data and peer-reviewed methodology.
DATA SOURCE
FDA OpenFDA API
All adverse event data is sourced live from the FDA OpenFDA FAERS database — the same public system used by researchers, regulators, and pharmaceutical companies worldwide.
METHODOLOGY
PRR + Isolation Forest
Signal detection uses Proportional Reporting Ratio, the EMA-endorsed standard, combined with Isolation Forest ML for temporal anomaly detection across quarterly reporting trends.
STORAGE
MongoDB Atlas
Reports and signals are stored using MongoDB Atlas — enabling fast aggregation across large-scale adverse event datasets with full document-level evidence traceability.
Important limitation: VigilanceAI signals indicate statistical correlation — not causation. FAERS data contains reporting biases. Signals should be interpreted by qualified healthcare professionals and not used for clinical decision-making.
